# cl-mcp Feedback

This directory is the canonical place to accumulate feedback about using
`cl-mcp` during implementation work in this repository.

## File Naming

- use one Markdown file per task or session
- prefer `YYYY-MM-DD-short-topic.md`
- update an existing file for the same task instead of creating duplicates

## What To Record

- tools used and the workflow that worked well
- friction points or failure modes
- workarounds that were effective
- concrete suggestions for improving `cl-mcp` or the local workflow

Keep entries concise and specific enough that later agents can reuse them.
