cl-acorn プロジェクト・コンセプト文書
〜 LLMエージェントと共創する、動的ホワイトボックス・モデリング基盤 〜

1. ビジョン：なぜこのアプローチが必要なのか
現代のデータサイエンスには、二つの大きなアプローチが存在します。

深層学習（ブラックボックス）: 膨大なデータと計算資源（GPU）を用いて、機械に内部構造（理屈）をゼロから学習させるアプローチ。複雑なタスクに強いが、解釈性が低く、データが少ない状況では機能しない。

確率的プログラミング（ホワイトボックス）: 人間が持つ「ドメイン知識（理屈）」を数理モデルとして記述し、少量のデータから未知のパラメータの不確実性を推論するアプローチ。解釈性が高く少データに強いが、人間がモデルを正しく設計する必要がある。

cl-acorn は、この「ホワイトボックス・アプローチ」の弱点であった「人間がモデルを設計しなければならない」という制約を、LLM（大規模言語モデル）の持つ膨大な事前知識と自律的エージェントループによって突破することを目指します。「Automated Statistician（自動統計家）」として、AI自身が仮説を立て、数式を書き、推論し、自己修正する次世代のデータモデリング環境を提供します。

2. cl-acornがCommon Lisp上で実装されている必然性
PPL（確率的プログラミング言語）の基盤として、C++でもPythonでもなく、あえてCommon Lispを採用するのには強力な技術的必然性があります。

「コンパイル待ち時間ゼロ」のREPL駆動開発
Stanなどの既存ツールは、モデルを記述するたびにC++へのトランスパイルとコンパイルが発生し、人間の思考やAIエージェントの試行錯誤（イテレーション）のループを分断してしまいます。Common Lispは「REPL（対話的実行環境）」を言語のコアに据えており、コードを書き換えた瞬間に即座に評価・実行が可能です。これにより、仮説検証のサイクルを極限まで高速化できます。

コードとデータの同一性（同図像性）とマクロ
LispのプログラムはLispのデータ構造（S式）そのもので表現されます。これにより、AIエージェントにとって「コードを生成し、解析し、部分的に書き換える」ことが他の言語よりも圧倒的に容易で安全です。また、強力なマクロ機能により、LLMが扱いやすいモデリング用のDSL（ドメイン特化言語）を言語の内部に自然に拡張・構築することができます。

ホスト言語の制御構造をそのまま微分可能（Define-by-Run）
専用の外部DSLを設けるのではなく、Lispネイティブの if、cond、dotimes などの制御構造をそのまま自動微分に通すことができます。これにより、AIエージェントは「シミュレーションのループ」や「複雑な条件分岐」を含むプログラムを、通常のコーディングと同じ感覚でモデル化できます。

3. cl-mcpによる「AIエージェントとの動的モデリング・実験」の意義
cl-acorn は、Lisp環境をAIエージェントに開放するMCPサーバーである cl-mcp と密接に連携することで、真価を発揮します。この組み合わせがもたらす意義は以下の通りです。

LLMのドメイン知識の直接的な実装（仮説生成）
LLMは物理学、疫学、経済学などの論文レベルの知識を持っています。AIエージェントは「この現象にはポアソン分布が適している」「この変化には微分方程式のこの項を追加すべきだ」という仮説を自律的に立案し、それを直接 cl-acorn のモデルコードとして記述できます。

エージェントによる「自律的な実験ループ」の実現
cl-mcp が提供するツール（repl-eval による即時実行や、lisp-edit-form による安全なコード編集）を利用することで、エージェントは以下のループを自律的に高速回転させます。

EXPLORE (探索): ドメイン知識に基づきモデルのプロトタイプを生成。

EXPERIMENT (実験): REPL上で推論を実行し、データとの適合度を評価。

REFINE (洗練): 予測誤差やMCMCの収束状況を分析し、コードを修正。

PERSIST (永続化): 成功したモデルをファイルに保存。

解釈可能な「理屈」の獲得
AIが最終的に出力するのは「ブラックボックスな重みパラメータ」ではなく、「Common Lispで書かれた数理モデルのソースコード」です。人間はそのコードを読み、AIがどのような理屈（メカニズム）でその予測に至ったのかを完全に解釈し、検証することができます。

4. そのために cl-acorn はどのような機能を持つべきか
AIエージェントが自律的に仮説検証ループを回すための「キャンバス」として、cl-acorn は以下の機能を備える必要があります。

堅牢で透過的な自動微分（AD）基盤

現在の前進モード（Forward-mode）に加え、多数のパラメータを効率的に扱うための後退モード（Reverse-mode / Backpropagation）のサポート。

ホスト言語のあらゆる制御構造やネイティブ関数を、エラーなく微分可能な状態で通過させるロバスト性。

AIに「失敗の理由」を的確に伝えるエラーハンドリング

数学的な不整合（非負の制約違反、確率の合計が1にならない等）や、勾配の消失・爆発が起きた際、単にクラッシュするのではなく、LLMが原因を特定してコードを修正（Refine）できるような「構造化されたエラーメッセージ」と「コンテキスト」をREPL経由で返す機能。Common Lispのコンディション・リスタートシステムをフル活用する。

推論アルゴリズムの部品化（コンポーザビリティ）

「ブラックボックスな全自動推論ボタン」を一つ用意するのではなく、HMC（ハミルトニアンモンテカルロ）、変分推論（VI）、勾配降下法、各種確率分布などの「部品（ビルディングブロック）」を提供する。これにより、AIエージェントがタスクに応じて柔軟に推論ロジックを組み立て、デバッグできるようにする。

状態や途中経過のイントロスペクション（内部観察）機能

エージェントが最適化の途中で「現在どのように分布が収束しているか」「どのパラメータの勾配が怪しいか」をREPL上で動的に問い合わせ（Inspect）できる機能。

総括:
cl-acorn と cl-mcp の組み合わせは、単なる「Lispで書かれた微分のライブラリ」ではありません。それは、**「汎用知能（LLM）が、データから解釈可能な科学的モデル（ホワイトボックス）を自動生成・検証するための、最高速度の思考エンジン」**として機能します。
